## Архитектура потока оплаты

```
Пользователь → Квиз → Paywall
    ↓ (кнопка Оплатить)
Apps Script: createPayment_()
    ↓ POST Init
Tinkoff Kassa → PaymentURL
    ↓ редирект
Tinkoff форма оплаты (карта/СБП)
    ↓ успех
SuccessUrl = klaviron.github.io/?payment=ok&order_id=XXX
    ↓
Фронтенд: verifyPayment(order_id)
    ↓ GET action=verify
Apps Script: проверка orders + Tinkoff GetState
    ↓ ok + token
Фронтенд: показ полного результата
```

## Безопасность
- SecretKey хранится в Script Properties (не в коде)
- Подпись проверяется на каждом webhook
- order_id = UUID, привязан к answers квиза
- Токен сессии = order_id (передаётся в URL после success)
