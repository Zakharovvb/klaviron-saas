## 08.07.2026 — Стратегия монетизации (план, отложено)

### Решение
- **Один домен** (klaviron.github.io → позже klaviron.ru), **одна кодовая база**
- **Две внутренние ветки монетизации**, переключение по URL/UTM, без пользовательского тумблера:
  - `?mode=paid` (по умолчанию) — платный результат 299 ₽
  - `?mode=market` (будущее) — бесплатный результат + партнёрские ссылки Яндекс.Маркета
- **Не делать два домена** — дробит SEO, аналитику, поддержку

### Почему платная версия сначала
1. Товар специфичный — покупатели идут в спец. магазины, не на маркетплейс
2. Яндекс.Маркет партнёрка требует модерацию (2-14 дней), живой ресурс, контент
3. Нет подтверждённого спроса на 299 ₽ — нужно проверить first
4. Партнёрка Яндекса ограничивает источники трафика (таргет/контекст на бренд)

### План запуска (по этапам)
| Этап | Что делаем | Зачем |
|------|-----------|-------|
| 1 | Довести платную версию 299 ₽ до конца | Проверить willingness to pay |
| 2 | Запустить трафик на один оффер | Не размывать аналитику |
| 3 | Собрать первые 10–30 оплат | Подтверждение спроса |
| 4 | Опубликовать 10+ материалов (сайт/Telegram) | Усилить доверие, подготовить модерацию |
| 5 | Податься в Яндекс.Маркет партнёрку | С живой площадкой |
| 6 | Добавить `?mode=market` как апселл/альтернативу | Вторая монетизация |

### Yandex.Market — конкретные ссылки под каждую модель
- Не одна общая ссылка, а партнёрская под каждую рекомендованную модель
- Кнопка «Открыть на Яндекс.Маркете» в результате квиза
- URL — партнёрская ссылка на товар или поисковую выдачу по модели

### Метрики для сравнения (когда будут обе ветки)
- **market:** EPC = клики × конверсия_в_покупку × комиссия
- **paid:** RPV = конверсия_в_оплату × 299 ₽
- Побеждает ветка с большей выручкой на 100 посетителей

### Безопасный стартовый сценарий
- 80% трафика → paid
- 20% трафика → market (тест)
- Через 2-3 недели сравнить: доход/100 посетителей, completion rate, CTR, жалобы

## МЕХАНИКА ПРОЦЕССА ОПЛАТЫ (end-to-end)

### Поэтапное описание:

**1. Пользователь проходит квиз**
- Отвечает на вопросы (goal, experience, format, budget и т.д.)
- На последнем шаге вызывается `submitQuiz()`

**2. Показ paywall**
- `PAYMENT_ENABLED = true` и `sessionToken = null` → показывается paywall
- `pickLocalResult()` определяет тип инструмента (из QuizEngine) и показывает его бесплатно
- Кнопка: «Получить подбор — 299 ₽»

**3. Нажатие кнопки «Получить подбор»**
- Вызывается `startPayment(payload)`
- Кнопка меняет текст на «Перенаправляем на оплату…»
- Фронтенд шлёт GET-запрос к GAS:
  `?action=createpayment&goal=hobby&budget=low&experience=beginner&format=synth&amount=29900`

**4. GAS создаёт платёж в ЮKassa**
- `createPaymentFromQuiz_()` генерирует `orderId = klv_XXXXXXXX...`
- Вызывает `yookassaCreatePayment_()` → POST на `https://api.yookassa.ru/v3/payments`
- ЮKassa возвращает `confirmation_url`
- GAS сохраняет заказ в Google Sheets (лист `orders`): order_id, status=NEW, goal, budget, amount
- Возвращает фронтенду: `{ ok: true, orderId, paymentUrl }`

**5. Редирект на ЮKassa**
- `window.location.href = data.paymentUrl` — пользователь попадает на страницу ЮKassa
- Пользователь вводит данные карты / выбирает СБП / SberPay
- Оплачивает 299 ₽

**6. ЮKassa обрабатывает платёж**
- При успехе ЮKassa шлёт **webhook** (POST) на GAS: `payment.succeeded`
- GAS обновляет статус в Sheets: `succeeded`
- Если SberPay (`waiting_for_capture`) → GAS автоматически подтверждает через `yookassaCapturePayment_()`
- ЮKassa редиректит пользователя обратно: `https://klaviron.ru/?payment=ok&order_id=klv_XXX`

**7. Возврат на сайт — проверка оплаты**
- `handlePaymentReturn()` срабатывает при загрузке
- Видит `?payment=ok&order_id=klv_XXX`
- Очищает URL (`replaceState`)
- Показывает спиннер «Проверяем оплату…»

**8. Verify с retry (новая логика)**
- Вызывается `verifyWithRetry(orderId, 3, 3000)`:
  - **Попытка 1**: GET `?action=verify&order_id=klv_XXX`
    - GAS проверяет в Sheets статус заказа
    - Если `succeeded` → возвращает `{ ok: true, token: orderId }`
    - Если нет → проверяет напрямую через ЮKassa API (`yookassaGetPayment_`)
    - Если платёж найден и подтверждён → обновляет Sheets, возвращает ok
  - Если ответ не ok или ошибка сети → ждёт 3 секунды, показывает «попытка 1 из 3»
  - **Попытка 2**: то же самое
  - **Попытка 3**: последняя попытка
  - Если все 3 неудачны → возвращает `null`

**9. Если оплата подтверждена (`data.ok === true`)**
- `sessionToken = data.token` (или orderId)
- Показывает экран «✓ Оплата подтверждена! Полный отчёт разблокирован»
- Кнопка «Смотреть результат»
- Нажатие → `submitQuiz()` снова, но теперь с `sessionToken`

**10. Загрузка платного результата**
- `submitQuiz()` шлёт `?action=paidResult` (вместо `previewResult`)
- GAS вызывает `getPaidResult_()`:
  - Читает каталог из Google Sheets (`readCatalogFromSheet_`)
  - Фильтрует модели по параметрам (бюджет, тип, формат)
  - Возвращает 3 модели с ценами и ссылками на **Яндекс Маркет**
  - Плюс комплектация, реальная цена, предупреждения
- Фронтенд рендерит полный результат

**11. Если оплата НЕ подтверждена**
- После 3 попыток → экран «Оплата не подтверждена после 3 попыток»
- Кнопка «Повторить оплату» → возврат к началу

### Возможные проблемы:
- **Webhook не пришёл** — retry через 3 сек + прямой запрос к ЮKassa API ловят платёж
- **Webhook не настроен** — `verifyPaymentServer_` имеет fallback: ищет платёж по order_id в metadata
- **DNS error в GAS** — try-catch во всех функциях с UrlFetchApp.fetch
